# Git入門ガイド（共著者向け）
> 「Dropboxの上位互換」として使う。これだけ読めば大丈夫。

---

## 1. Gitとは何か（30秒で理解）

Dropboxと同じく**ファイルの同期ツール**。ただし：

- 「いつ・何を・なぜ変えたか」が記録に残る
- 過去の任意のバージョンにいつでも戻せる
- ファイル名に `_v2` `_最終版` `_本当の最終版` をつける必要がない

---

## 2. 毎日のワークフロー（これだけ覚える）

```
作業を始める前
  → git pull

作業が終わったら
  → git add .
  → git commit -m "〇〇を修正"
  → git push
```

**具体例：**

```bash
# 作業開始時（必ず最初にやる）
git pull

# --- ここで普通にファイルを編集 ---

# 作業終了時
git add .
git commit -m "第3節の推定結果を更新"
git push
```

---

## 3. コマンド一覧（5つだけ）

| コマンド | 意味 | いつ使う |
|---------|------|---------|
| `git pull` | 最新版を取得 | **作業開始前・必ず** |
| `git add .` | 変更をまとめる | コミット前 |
| `git commit -m "説明"` | スナップショットを保存 | 作業の区切りごと |
| `git push` | GitHubに送信 | **作業終了後・必ず** |
| `git status` | 今の状態を確認 | 迷ったとき |

---

## 4. コンフリクト（衝突）が起きたら

二人が同じ箇所を同時に編集すると衝突が起きる。**データは消えない。怖くない。**

衝突すると、ファイルの中に次のような表示が出る：

```
<<<<<<< HEAD
あなたの変更内容
=======
相手の変更内容
>>>>>>> main
```

**対処法：**
1. `<<<<<<<`〜`>>>>>>>`の部分を手動で正しい内容に書き直す
2. `git add .` → `git commit -m "コンフリクト解決"` → `git push`

**迷ったら担当者（Daisuke）に連絡してください。**

---

## 5. よくある間違いとその対処

| 状況 | 対処 |
|------|------|
| `git pull` を忘れて作業した | `git pull` してからコミット。衝突したら上記の手順 |
| `git push` し忘れて翌日に気づいた | そのまま `git push` でOK |
| 何か壊したかも | **何もしないで** Daisukeに連絡 |
| `commit -m` のメッセージが英語か日本語か迷う | どちらでも。とにかく書く |

---

## 6. 絶対にやってはいけないこと

- ❌ データファイル（`.dta`, `.csv`, `.xlsx`）を **`git add`しない**（`.gitignore` で除外済み）
- ❌ パスワードや認証情報をファイルに書いてコミットしない
- ❌ 衝突したときに自己判断でリポジトリを削除しない

```
【確認】      git status

```
